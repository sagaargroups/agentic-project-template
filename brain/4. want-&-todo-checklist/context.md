# 📋 Want & To-Do Checklist Context

> **Folder Purpose:** The strategic backlog. This folder holds feature requests, bug trackers, future architectural plans, and "want-to-build" lists.

## 1. Allowed Contents
- Markdown files with task checklists (`[ ]`, `[x]`)
- Feature request outlines
- Prioritized bug reports
- Epics and user stories for future sprints

## 2. Operational Rules
- **Prioritization:** Maintain clear prioritization (High, Medium, Low). The founder dictates priority.
- **Status Tracking:** Keep task statuses updated. When a feature is completed, check it off the list immediately to keep the AI agents synchronized.

## 3. AI Guardrails
- **NEVER** start working on or executing code for an item on the "want" list without explicit authorization from the founder. This is a queue, not a command line.
- **ALWAYS** check this folder before starting a new session to see if there are any high-priority pending items that require immediate attention.
