# Module 4: The Execution Protocol

> **The plug-and-play layer. Run these checklists before, during, and after ANY work. Every unticked box = work remaining.**

---

## Phase A: PRE-FLIGHT (Before Work Begins)

> Time: Run once. Takes 30-60 minutes. Saves weeks of wasted effort.

### Foundation (Source A — What Others Built)
```
□ A01. [Engine A] I have studied 3-5 best examples of this done by others
□ A02. [Engine A] I have extracted their structure and adopted proven patterns
□ A03. [Engine A] My tool stack is locked — not building what already exists
□ A04. [Engine A] I understand WHY their structures work (not just WHAT)
```

### Experimentation (Hunting Unknowns)
```
□ A05. [Engine B] I have listed 3-5 things I DON'T KNOW that could kill this
□ A06. [Engine B] Each unknown has a micro-experiment (≤1 week)
□ A07. [Engine B] Experiments start BEFORE plan is "complete"
□ A08. [Engine B] Failure budget is set: 10-15% of total time/resources
```

### Inversion (Map Failure First)
```
□ A09. [Law 1] I have answered: "How would this DEFINITELY fail?"
□ A10. [Law 1] I am not doing any of the 6 silent killers
□ A11. [Law 1] The uncomfortable thing I'm avoiding is identified → THAT's my first action
```

### Questioning (Right Questions Loaded)
```
□ A12. [OS-Q] I have distinguished what I KNOW vs what I ASSUME
□ A13. [OS-Q] I have asked "WHY does this problem exist?" at least 3 levels deep
□ A14. [OS-Q] I have asked "What would someone from a different field do here?"
```

### Creativity (Differentiation Prepared)
```
□ A15. [OS-C] I have set at least 1 deliberate constraint
□ A16. [OS-C] I have identified my signature element — what only I would do
□ A17. [OS-C] I have at least 1 idea borrowed from an unrelated domain
```

### Speed (Iteration Engine Ready)
```
□ A18. [Law 2] My first version ships in 48 hours or less
□ A19. [Law 2] Feedback will reach me in hours/days — not weeks
□ A20. [Law 6] 90% safe foundation + 10% aggressive experiments (barbell set)
```

## Phase B: IN-FLIGHT (During Work)

> Time: Review every 48 hours minimum. Keep this visible while working.

### Engines Running
```
□ B01. [Engine A] Knowledge pipeline is active: I'm filtering, structuring, connecting — not just absorbing
□ B02. [Engine B] Achievement pipeline is active: I'm shipping, reflecting, documenting — not just planning
□ B03. [Flywheel] Knowledge is feeding Achievement AND vice versa — not running as parallel tracks
□ B04. [Engine B] I am practicing at the 85% success rate — challenged but not overwhelmed
```

### Experiments Active
```
□ B05. [Engine B] Experiments are RUNNING — I'm not waiting for "more data"
□ B06. [Engine B] I'm documenting what experiments teach (not just results)
□ B07. [Engine B] Experiment results are feeding back into my foundation
```

### Questions Deepening
```
□ B08. [OS-Q] I'm asking Q3+ level questions (causal/systemic/generative — not binary)
□ B09. [OS-Q] I have used inversion questions during execution ("What am I NOT doing?")
□ B10. [OS-Q] I have asked at least 1 adjacent question ("What would a [X] do?")
```

### Creativity Producing
```
□ B11. [OS-C] At every decision point: "Is this the obvious choice or MY choice?"
□ B12. [OS-C] My constraints are producing output I wouldn't create without them
□ B13. [OS-C] I have broken the problem down in at least 2 ways (Breakdown Reframe)
```

### Speed Maintained
```
□ B14. [Law 2] My OODA cycle is days, not weeks
□ B15. [Law 2] Slow experiments killed or redesigned within 7 days
□ B16. [Law 7] If reality contradicts my plan, I'm updating the PLAN (not forcing reality)
```

### Laws Applied
```
□ B17. [Law 5] If something went wrong, it TAUGHT me (antifragile response)
□ B18. [Law 4] I've imported at least 1 insight from another domain (adjacent possible)
```

## Phase C: POST-FLIGHT (After Work Completes)

> Time: Run once after completion. This is what separates one-time workers from compounders.

### Deliverables
```
□ C01. [Engine B] All tangible outputs are complete and shipped
□ C02. [OS-C] The output has my creative signature — it is NOT generic
□ C03. [OS-C] I can explain in 1 sentence what makes this unique
```

### Knowledge Audit
```
□ C04. [Engine A] My Knowledge level for this domain has increased — I can name specifically what I now know
□ C05. [Engine A] I know at least 1 thing the textbooks DON'T tell you (from real-world experience)
□ C06. [Truth 5] I can teach what I learned to someone else — clearly
```

### Achievement Audit
```
□ C07. [Engine B] I can name the Achievement level: Personal / Professional / Market / Industry
□ C08. [Engine B] Results are documented — decisions, outcomes, failures, surprises
□ C09. [Truth 8] Failures are documented with ROOT CAUSES, not just "it didn't work"
```

### Adaptation (System Evolution)
```
□ C10. [OS-A] HARVEST: Key learnings from this cycle are extracted
□ C11. [OS-A] INTEGRATE: The CPS has been updated with new patterns/anti-patterns
□ C12. [OS-A] PRUNE: Anything that didn't serve this work is simplified/removed
□ C13. [OS-A] EVOLVE: My benchmark for next time is HIGHER than this time
□ C14. [OS-A] My iteration speed target for next cycle is FASTER than this cycle
```

## Domain Application Matrix

> How all modules apply across different types of work:

| Domain | Foundation (Copy) | Experiment (Hunt Unknowns) | Creativity (Differentiate) | Questions (Right Ones) | Adapt (Evolve) |
|---|---|---|---|---|---|
| **Business** | Copy proven models, tools, SOPs | Test pricing, channels, audience in micro-bets | Unique positioning, brand voice, customer experience | "Why would someone PAY for this?" (depth) | Update ICP, pricing, messaging after each campaign |
| **Software** | Use proven architecture, libraries, patterns | Prototype risky features first, load-test assumptions | Unique UX, feature innovation, developer experience | "What would a GAME DESIGNER do with this UI?" (adjacent) | Post-mortem every sprint, update architecture decisions |
| **Research** | Follow established methodology, cite existing work | Run edge-case experiments others skip | Novel hypothesis, unique framework, new lens | "Why has nobody tested THIS variable?" (hidden variable) | Update methodology based on each study's surprises |
| **Learning a Skill** | Best courses, best teachers, proven curricula | Practice in uncomfortable scenarios, deliberate difficulty | Develop your own technique, your own style | "What am I avoiding practicing because it's hard?" (inversion) | After each practice block, update weakness map |
| **Content** | Study top performers in niche, adopt formats that work | Test hooks, formats, angles weekly | Your voice, your storytelling, your perspective | "What does my audience FEEL, not just think?" (depth) | Update content playbook after each batch of posts |
| **Client Work** | Adopt industry SOPs for delivery, use templates | A/B test strategies before committing client budget | Creative campaigns with your signature methodology | "How would this fail for THIS specific client?" (inversion) | Post-project review, update client playbook |
