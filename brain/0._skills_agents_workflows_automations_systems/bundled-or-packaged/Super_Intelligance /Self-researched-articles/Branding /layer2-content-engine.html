Content & Marketing

CONTENT
ENGINE

One idea enters. The system discovers every format it can become, distributes across every channel, measures what worked, and feeds that learning back. Automatically. Forever.

The Parent Rule: Every agent must do its core job, then self-discover what else it can do with its output, maximise value, and pass the enriched package to the next agent. Never stop at the minimum.

01
How the Engine Thinks
The 4 steps every input goes through
Step 01
UNDERSTAND
What is this content about? Who does it help? Which human gap does it fill — Belief, Capability, Utility, or Identity? This anchors every format that follows.
Step 02
CREATE CORE
Build the long-form foundation — a full blog post or article, SEO-optimised, structured, with every key point developed. Everything else is extracted from this.
Step 03
SELF-DISCOVER
What can I extract from this? Who else needs this? What format serves each platform best? The engine asks these before creating a single output piece.
Step 04
PACKAGE + PASS
All outputs bundled with metadata — keywords, hashtags, best posting times, audience segment, CTA per format — then handed to the Scheduler Agent.

03
What the Agent Asks Itself
The metadata layer applied to every output before passing to Scheduler
Question Why It Matters
What keywords make this rank?Determines SEO metadata attached to blog and Pinterest outputs. Without this, content is invisible to search.
What hashtags maximise reach?Calculated per platform — Instagram uses different logic than LinkedIn or X. Not one-size-fits-all.
What time does each platform perform best?Passed to Scheduler Agent as posting time recommendations per audience timezone and platform peak.
Which audience segment does each piece serve?Some outputs target cold audiences (SEO blog), others warm leads (email). Segmentation is labelled per output.
What is the CTA for each format?CTA changes per channel. Blog → subscribe. Story → swipe up. Email → click. Thread → follow. Each is set explicitly.
What content gap does this fill?Cross-checked against content calendar to avoid repetition and ensure full topic coverage over time.

06
The Parent Rule for All Agents
The thinking pattern every agent in the system must follow
1
RECEIVE
Accept the input. Understand it fully before acting on it.
2
DO YOUR JOB
Execute your core function completely and with quality.
3
SELF-DISCOVER
Ask: what else can this output become? Who else does it help?
4
MAXIMISE
Extract every unit of value from what was created.
5
ENRICH + PASS
Bundle with metadata. Hand enriched package to next agent.
6
NEVER MINIMUM
The system never stops at doing the least. It always finds more.
