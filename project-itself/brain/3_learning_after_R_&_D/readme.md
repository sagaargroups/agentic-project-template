# 🔬 Research & Learning

> **Your project's long-term memory.**

## What Lives Here
This is where your project **gets smarter over time**. Every piece of research, every lesson learned, every tech evaluation — it all accumulates here, creating a persistent knowledge base that both you and your AI agent can reference.

### Examples of What to Store
- 📊 Competitor analysis breakdowns
- ⚖️ Technology comparison reports (e.g., "Next.js vs Remix")
- 🐛 Lessons learned from bugs and failures
- 📈 SEO/GEO strategies and findings
- 📚 Article summaries and conference notes
- 🔍 Market research and user feedback analysis

## How to Use This Folder
1. After researching a topic, create a clearly named `.md` file (e.g., `auth-providers-comparison.md`)
2. Synthesize key findings into bullet points — don't just paste raw articles
3. Always include source links and a date stamp
4. Your AI will cross-reference this folder before proposing new tech decisions

## Why This Matters
Projects that learn from their own history make fewer repeated mistakes. This folder is the difference between an AI that starts from scratch every session and one that compounds knowledge over months.
