# 📋 Tasks & Daily Logs

> **Your project's activity ledger and backlog.**

## What Lives Here
This is your project's **task management system and daily journal**. It tracks what needs to be done, what's in progress, and maintains a historical log of all contributions — by both humans and AI agents.

### Examples of What to Store
- ✅ Task checklists and to-do lists
- 🐛 Bug reports with priority levels
- 💡 Feature requests and enhancement ideas
- 📓 Daily contribution logs (what was built each day)
- 🗓️ Sprint planning documents

## How to Use This Folder

### Task Lists
Create task files with clear priorities:
```markdown
# Feature: User Authentication
- [x] 🔴 Set up Firebase Auth
- [ ] 🟡 Add Google OAuth
- [ ] 🟢 Add password reset flow
```

### Daily Logs
After each working session, append a daily log entry:
```markdown
## 2026-06-22
### ✅ Completed
- Restructured brain/ directory
- Created all context.md files

### 🔜 Next
- Write the public README.md
```

## Why This Matters
Daily logs create accountability and give your AI agent a complete timeline of the project's evolution. When an AI reads these logs, it understands not just *what* the code does, but *when* and *why* each decision was made.
